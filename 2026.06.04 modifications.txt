Here is the list of improvements and bug fixes applied to the repository to support remote Ollama endpoints and resolve the embedding model context limit errors:

#### 1. Remote and External Server Support (Ollama Health Checks)

- **Problem**: The health check was checking a hardcoded localhost `/health` endpoint. For Ollama (which does not expose `/health`), this failed, causing the scripts to try to launch the Hugging Face local server on port 11434, resulting in address binding conflicts (`Errno 10048`).
- **Fix**: Rewrote the server health checks in `11_run_hybridqa_proper_compare.py` and the baseline/GraphRAG smoke tests. The check now:
    - Dynamically extracts the target host/port from the `LOCAL_LLM_BASE_URL` (supporting remote servers like `espadon1`).
    - Checks standard endpoints (`/` and `/v1/models` in addition to `/health`) to auto-detect already-running Ollama and OpenAI-compatible instances.
    - Bypasses starting the local server when an external model service is detected.

#### 2. Configurable Chunking Size for Embedding Compatibility

- **Problem**: The GraphRAG chunk size was hardcoded to `4000 tokens`. When indexing large HybridQA files, these chunks exceeded the physical context window limit (`512 tokens`) of the quantized BERT-based embedding model (`jeffh/intfloat-e5-base-v2:q8_0`), leading to index crashes.
- **Fix**: Added dynamic chunking size settings in `src/config/settings.py` and `src/graphrag_system/runner.py`:
    - Configured a default chunk size of **`300 tokens`** (matching GraphRAG's standard default), which successfully fits within the embedding model's 512-token limit.
    - Exposed `GRAPHRAG_CHUNK_SIZE` and `GRAPHRAG_CHUNK_OVERLAP` env variables in the `.env` configuration file to allow for future adjustments.

#### 3. Client Timeout Enforcement to Prevent Hanging

- **Problem**: The `openai` client instantiation had no default timeout configured. When Ollama was not running or took a long time to load a model, API requests would hang indefinitely, locking up the python processes.
- **Fix**: Modified `src/utils/model_client.py` and `src/evaluation/ragas_support.py` to enforce a standard **60-second timeout** on all generation and embedding requests. The scripts will now fail with a visible error instead of hanging.
